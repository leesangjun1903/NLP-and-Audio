## 1. 핵심 주장과 주요 기여 요약

### 1.1 핵심 주장

TSLM의 핵심 주장은 다음 두 가지로 압축된다:

> **"언어 모델이 순차적으로 추론을 생성하는 구조적 한계를 극복하기 위해, 트리 구조 탐색을 단일 생성 과정 내에 내재화(internalize)할 수 있으며, 이는 강화학습이나 외부 오케스트레이션 없이도 달성 가능하다."**

기존 LLM의 순차 생성 패러다임:

$$p(y \mid x) = \prod_{t=1}^{|y|} p(y_t \mid x, y_{ < t})$$

이 방식은 단일 경로에 선형적으로 커밋되므로, 복잡한 문제에서 다중 탐색 경로를 체계적으로 병렬 탐색하지 못한다. TSLM은 이를 트리 구조의 내재적 생성으로 해결한다.

### 1.2 주요 기여

| 기여 | 설명 |
|------|------|
| **TSLM 프레임워크** | 특수 토큰 기반 직렬화로 트리 구조 추론을 SFT로 학습 |
| **성능 향상** | Game of 24: 100% (SC 17%), Gridworld OOD: 91.5% (ToT@100 42.7%) |
| **창발적 능력** | 풀 수 없는 문제 97% 자동 식별 (기존 방법 0%) |
| **새로운 추론-시간 스케일링** | $k$개 독립 샘플링 대신 단일 일관된 트리 내 $k$개 분기 탐색 |
| **일반화 성능** | 훈련 분포(10×10 Gridworld) 외 환경(20×20)으로 강건한 외삽 |

## 2. 상세 설명

### 2.1 해결하고자 하는 문제

기존 LLM 기반 추론 시스템의 문제는 4가지로 분류된다:

1. **단일 경로 선형 커밋**: 순차 생성 구조상 초기 실수에서 회복 불가
2. **오류 전파(Error Propagation)**: 초기 단계 오류가 전체 경로를 망침
3. **중복 계산**: 여러 경로에서 공유 프리픽스를 반복 계산
4. **병렬 탐색 불가**: 구조적으로 분기 탐색을 동시에 진행할 수 없음

Tree-of-Thought(ToT)와 같은 외부 탐색 방법은 이를 보완하지만:
- **독립 샘플링**으로 인해 탐색 공간의 일부가 중복되거나 누락됨
- **여러 번의 독립 forward pass** 필요 → 계산 비효율
- **훈련과 추론이 분리**되어 있어 외부 복잡도 증가 시 일반화 실패

### 2.2 제안 방법

#### 2.2.1 다중 후계 상태 모델링

현재 상태 $s$에서 단일 행동 $a$를 예측하는 대신, TSLM은 복수의 후계 상태를 생성한다:

$$\pi_\theta(s) = \{T(s, a_1), \cdots, T(s, a_k)\}$$

여기서 $a_i$는 각 분기, $k$는 분기 인수(branching factor)이다. 각 $a_i$는 동일 노드의 이전 행동 $a_1, \cdots, a_{i-1}$에 조건부로 생성되어, **체계적인 탐색 공간 커버리지**를 보장한다.

#### 2.2.2 토큰 기반 트리 직렬화

트리 구조를 선형 시퀀스로 변환하기 위해 특수 토큰 집합을 도입한다:

| 토큰 | 의미 |
|------|------|
| `[SEP]` | 더 탐색 가능한 유효 행동 |
| `[FAIL]` | 더 이상 탐색 불가한 막힌 경로 |
| `[GOAL]` | 목표 상태 도달 |
| `[BOS]` / `[EOS]` | 각 노드의 자식 탐색 시작/끝 |

이 직렬화는 **성공 경로와 실패 경로를 모두 포함**하여 모델이 완전한 탐색 과정을 학습한다.

#### 2.2.3 학습 목적 함수

**단순 학습 손실 (전체 직렬화 시퀀스에 대한 표준 LM 손실):**

$$\mathcal{L} = -\sum_{t=1}^{T} \log p(y_t \mid y_{ < t}, x) $$

**형식적 학습 목적 (트리 구조를 명시적으로 반영):**

탐색 트리 $\mathcal{T}$의 노드 집합 $\mathcal{N}$에 대해, 노드 $s_i$의 컨텍스트는:

$$\text{ctx}(s_i) = \bigcup_{s_j \in \pi(s_i)} y_{s_j} \cup \bigcup_{\substack{s_k \in \text{siblings}(s_i) \\ k < i}} y_{s_k} $$

- $\pi(s_i)$: 루트에서 $s_i$까지의 경로 (조상 노드들)
- $\text{siblings}(s_i)$: 동일 부모를 공유하는 형제 노드 (좌→우 순서로 이전에 생성된 것)

이를 바탕으로 전체 학습 목적함수:

$$\mathcal{L} = -\frac{1}{|\mathcal{N}|} \sum_{s_i \in \mathcal{N}} \sum_{t=1}^{|y_{s_i}|} \log p(y_{s_i, t} \mid y_{s_i, < t}, \text{ctx}(s_i)) $$

이 수식의 핵심은 각 노드가 **(1) 조상 경로**와 **(2) 이전에 생성된 형제 노드**에만 조건부로 학습되며, **관련 없는 서브트리에는 조건부가 걸리지 않는다**는 것이다 → **선택적 컨텍스트 분리(Context Decoupling)**.

#### 2.2.4 추론 절차

1. 현재 상태에서 다중 후보 행동 생성
2. `[SEP]`, `[FAIL]`, `[GOAL]` 토큰 파싱으로 분기 식별
3. 유효 분기(`[SEP]`)를 독립 시퀀스로 포크(fork) → "Stitching" 과정
4. 각 유효 상태를 `[GOAL]` 발견 또는 탐색 소진까지 재귀적 확장
5. 기본 탐색 전략: BFS(너비 우선 탐색); 대안으로 DFS 사용 가능

#### 2.2.5 오픈 엔드 태스크를 위한 의사 트리 생성 (Algorithm 1)

명시적 트리가 없는 태스크의 경우 **Guided Search Tree Bootstrapping**을 적용:

- 감독 모델 $\pi_\theta$로 각 상태에서 후보 행동 빔 서치 샘플링
- 알려진 정답(gold trajectory)을 우선순위 분기로 포함
- 보상 함수 $R(s, a)$ 기반 정렬로 유망 분기 우선화
- 중복 경로 제거(deduplication)

### 2.3 모델 구조

TSLM은 **새로운 아키텍처를 도입하지 않으며**, 표준 트랜스포머(Llama-3-8B, Llama-3-8B-Instruct, Qwen-2.5-7B 등)를 베이스로 사용한다. 혁신은 아키텍처가 아닌 **학습 데이터 포맷과 학습 목적함수의 설계**에 있다:

```
[기존 Sequential LM]
Input → Token₁ → Token₂ → ... → TokenN (단일 직선 경로)

[TSLM]
Input → [BOS] Action₁[SEP] Action₂[FAIL] ... [EOS]
              ↓
         Action₁ → [BOS] SubAction₁[SEP] SubAction₂[GOAL] [EOS]
         (트리 구조를 DFS 순서로 직렬화하여 표준 트랜스포머로 학습)
```

**핵심**: 시퀀스는 선형이지만, **조건부 의존성은 트리 토폴로지를 따른다**.

### 2.4 성능 향상

| Task | SC (pass@1) | PC (pass@1) | GRPO (pass@1) | **TSLM (pass@1)** | ToT (pass@100) |
|------|:-----------:|:-----------:|:-------------:|:-----------------:|:--------------:|
| Game of 24 | 17.0% | 47.0% | 15.0% | **100%** | 32.0% |
| Gridworld (i.d. 10×10) | 78.2% | 99.7% | 24.0% | **100%** | 95.0% |
| Gridworld (o.o.d. 20×20) | 33.0% | 81.1% | 6.0% | **91.5%** | 42.7% |
| ProntoQA | 99.7% | 97.5% | 99.8% | **100%** | 100% |
| GSM8K | 55.8% | 55.9% | 60.8% | **61.6%** | 62.3% |

> ⚠️ **중요한 비교 맥락**: ToT는 pass@100(100번 시도 중 1번이라도 성공), 나머지는 pass@1(1번 시도). 이 불리한 조건에서도 TSLM이 대부분 태스크에서 ToT를 능가한다.

**추론 효율성**: TSLM은 ToT 대비 유의미하게 빠른 벽시계 시간(wall-clock time)을 보임. 공유 프리픽스 재계산 없이 단일 forward pass로 트리 구성 가능.

### 2.5 한계점

| 한계 | 설명 |
|------|------|
| **계산 오버헤드 (학습)** | 전체 트리 노드 학습 → 평균 트리 크기 배수의 계산 비용 증가 |
| **감독 모델 품질 의존성** | 합성 트리 생성 품질이 감독 모델 역량에 크게 의존 (Llama-3-8B는 GSM8K 트리 품질 낮음) |
| **오픈 엔드 도메인 한계** | 창의적 글쓰기 등 주관적 정답이 있는 도메인에서 분기 유효성 판단 기준 설정이 어려움 |
| **비교 범위** | RAP, LATS, Reflexion 등 온라인 계획 방법과의 직접 비교 미수행 |
| **훈련 데이터 규모** | 태스크당 10K 미만 데이터로 실험 → 대규모 설정에서의 검증 미흡 |

## 3. 모델의 일반화 성능 향상 가능성

이 논문에서 **일반화 성능 향상**은 가장 중요하고 주목할 만한 발견 중 하나이다.

### 3.1 훈련 분포 외 복잡도 외삽 (OOD Generalization)

**핵심 실험**: 10×10 Gridworld로 훈련 후 20×20 Gridworld 테스트

| 방법 | 10×10 (i.d.) | 20×20 (o.o.d.) | 성능 저하율 |
|------|:------------:|:--------------:|:-----------:|
| SC | 78.2% | 33.0% | **−57.8%p** |
| PC | 99.7% | 81.1% | −18.6%p |
| **TSLM** | **100%** | **91.5%** | **−8.5%p** |
| ToT@100 | 95.0% | 42.7% | **−55.3%p** |

TSLM은 훈련 데이터보다 복잡한 환경에서도 **성능 저하가 가장 작다**. 논문에서는 이를 다음과 같이 해석한다:

> *"TSLM의 구조적 트리 표현이 특정 구성을 암기하는 것이 아닌, 일반화 가능한 탐색 패턴을 학습하기 때문이다."*

이는 **외부 탐색 알고리즘(ToT)도 복잡도 증가 시 일반화에 실패**함을 보여주며, 탐색 절차가 훈련 중 내재화되어야 한다는 주장을 뒷받침한다.

### 3.2 분기 인수 외삽

훈련 시 $k=5$로 설정했지만, 추론 시 $k=10$으로 확장하면:

| 분기 인수 | 수렴 정확도 |
|:---------:|:-----------:|
| $k=5$ (훈련) | 67.2% |
| $k=10$ (추론 시 외삽) | **71.1%** |

이는 모델이 학습된 탐색 전략을 **더 넓은 탐색 공간으로 일반화**할 수 있음을 의미한다. 단순히 더 많은 독립 샘플을 뽑는 병렬 샘플링과 달리, TSLM은 탐색 전략 자체를 일반화한다.

### 3.3 다양한 베이스 모델에서의 일관성

Llama-3-8B, Llama-3-8B-Instruct, Qwen-2.5-7B 모두에서 TSLM이 일관되게 기준선을 능가한다 (Appendix §E.2). 이는 일반화 이점이 **특정 모델 아키텍처 의존이 아닌 구조적 탐색 방법론**에서 비롯됨을 시사한다.

### 3.4 풀 수 없는 문제 인식 (Emergent Generalization)

TSLM은 훈련 중 풀 수 없는 문제를 명시적으로 학습하지 않았음에도 불구하고:

- 100개의 풀 수 없는 Game of 24 인스턴스 중 **97개(97%) 정확히 식별**
- SC, PC, GRPO: **0개 식별** (모두 잘못된 답 생성 = 환각)

이는 완전한 탐색 트리를 내재화함으로써 **훈련 분포에 없는 "해 없음" 케이스에 자연스럽게 일반화**되는 창발적 능력이다.

### 3.5 일반화 메커니즘 분석

일반화 성능 향상의 구조적 이유:

```
[ToT/SC 일반화 실패 원인]
복잡도 증가 → 탐색 공간 지수적 증가
→ 외부 알고리즘이 증가된 공간을 효율적으로 처리 못함
→ 성능 급락

[TSLM 일반화 성공 원인]
완전한 탐색 트리 학습
→ 상태 공간 탐색의 일반적 패턴 (분기, 백트래킹, 종료 인식) 내재화
→ 더 큰 공간에서도 동일 패턴 적용 가능
→ 성능 유지
```

**수식적 관점**: 식 (2)-(3)의 선택적 컨텍스트 조건부화( $\text{ctx}(s_i)$ )가 핵심이다. 각 노드는 전체 탐색 히스토리가 아닌 **관련 조상과 형제**에만 조건부로 학습되므로, 더 깊거나 넓은 트리에서도 동일한 조건부 패턴을 활용할 수 있다.

## 4. 앞으로의 연구에 미치는 영향과 고려 사항

### 4.1 앞으로의 연구에 미치는 영향

#### 4.1.1 패러다임 전환: SFT의 재평가

TSLM의 가장 중요한 함의는 **"SFT는 암기, RL은 일반화"** (Chu et al., 2025)라는 통념에 대한 반론이다:

- GRPO(RL 기반): Game of 24에서 15% 달성
- TSLM(SFT 기반): 100% 달성

이는 **학습 데이터의 구조(완전한 탐색 트리 vs. 최종 답)**가 학습 방법(SFT vs. RL)보다 더 중요할 수 있음을 시사한다.

#### 4.1.2 추론-시간 스케일링(Inference-Time Scaling)의 새로운 방향

| 기존 패러다임 | TSLM 패러다임 |
|:------------:|:------------:|
| $k$개 독립 샘플 생성 | 단일 트리 내 $k$개 분기 탐색 |
| "더 많이 샘플링" | "더 전략적으로 탐색" |
| 공유 프리픽스 중복 계산 | 공유 프리픽스 1회 계산 |
| 비조율적 병렬 탐색 | 조율된 일관성 있는 탐색 |

이는 **추론 컴퓨팅 예산 배분 연구**에 새로운 차원을 열어준다.

#### 4.1.3 환각(Hallucination) 감소 메커니즘

TSLM의 풀 수 없는 문제 인식 능력은 **LLM 환각 문제의 새로운 접근법**을 제시한다. 기존에는 RL 기반 후처리로 거부 행동을 학습했으나, TSLM은 완전한 탐색을 통해 자연스럽게 이를 학습한다.

### 4.2 앞으로 연구 시 고려할 점

#### 4.2.1 기술적 고려 사항

**① 계산 효율화 (학습 비용 문제)**

훈련 비용이 트리 크기 배수로 증가하므로:
- **Tree-aware Attention Caching**: 공유 조상의 KV 캐시 재활용 메커니즘 개발
- **Sparse Gradient Updates**: 핵심 노드에만 기울기 업데이트 집중
- **Curriculum Learning**: 작은 트리 → 큰 트리로 점진적 학습

**② 트리 품질 보증 (Supervision Quality)**

합성 트리 생성의 품질이 결과에 직접 영향을 미치므로:
- 감독 모델 선택을 자동화하는 메트릭 개발 필요
- 트리 품질 검증 프레임워크 (다양성, 커버리지, 정확성 균형)
- 더 나은 보상 함수 $R(s, a)$ 설계 (현재 RAP 방식 사용)

**③ 오픈 엔드 도메인 적용**

이진 검증이 없는 도메인(창의적 글쓰기, 오픈 대화)을 위해:
- **Soft Viability Signal**: 이진 [SEP]/[FAIL] 대신 연속 확률 기반 분기 판단
- **Preference-based Tree Construction**: RLHF 방식의 선호도 기반 트리 구성
- **LLM-as-Judge**: 대형 모델을 심판으로 활용한 분기 유효성 평가

**④ 토큰 효율성 문제**

완전한 탐색 트리를 직렬화하면 시퀀스 길이가 크게 증가한다. 특히 Game of 24의 PC 인스턴스에서 볼 수 있듯이, PC 방식은 매우 긴 시퀀스를 생성한다. 이를 위해:
- **Hierarchical Compression**: 트리의 일부를 압축 표현으로 대체
- **Dynamic Context Window Management**: 관련 없는 서브트리 컨텍스트 창에서 제거

#### 4.2.2 방법론적 고려 사항

**⑤ 더 강력한 기준선과의 비교**

논문은 RAP, LATS, Reflexion 등 더 발전된 외부 탐색 방법과의 비교를 수행하지 않았다. 향후 연구에서:
- 온라인 계획(LATS, Reflexion)과 오프라인 계획(TSLM)의 **혼합 접근법** 가능성 탐색
- MCTS 기반 방법과의 공정한 계산 예산 비교

**⑥ 강화학습과의 결합**

논문 자체도 언급하듯이:
- **ReJump** (Zeng et al., 2025): 더 나은 감독 모델로 트리 생성 개선
- **SoS** (Gandhi et al., 2024): 외부 탐색과 TSLM의 자기 학습 결합
- TSLM으로 초기화 후 RL fine-tuning으로 추가 개선 가능성

**⑦ 평가 방법론 표준화**

pass@1 vs. pass@100 비교의 비대칭성을 해소할 표준화된 평가 프로토콜 필요:
- 계산 예산 정규화(FLOP 수 기준 비교)
- 다양한 $k$에서의 성능 프로파일 전체 보고

#### 4.2.3 이론적 고려 사항

**⑧ 언제 TSLM이 유리한가?**

현재 실험 결과로부터 추론할 수 있는 조건:
- 명확한 탐색 공간이 있는 구조적 문제: TSLM 크게 유리
- 정답 검증이 명확한 수학/논리 문제: 유리
- 단순 정보 검색 태스크: 이점 미미

향후 연구는 **TSLM 적용 이점의 이론적 조건**을 명확히 해야 한다.

| 연구 | 발표 | 접근 방식 | 탐색 방법 | 일반화 | 계산 효율 | TSLM과의 차이 |
|------|:----:|:--------:|:--------:|:------:|:---------:|:------------|
| **Chain-of-Thought** (Wei et al., 2022) | 2022 | SFT/프롬프팅 | 없음 (선형) | 낮음 | 높음 | 단일 경로만 학습 |
| **Self-Consistency** (Wang et al., 2023) | 2023 | 샘플링 앙상블 | 독립 병렬 샘플링 | 중간 | 낮음 | 비조율적 독립 샘플 |
| **Tree-of-Thoughts** (Yao et al., 2023) | 2023 | 외부 탐색 | BFS/DFS 외부 오케스트레이션 | **낮음** (OOD 42.7%) | **낮음** (중복 계산) | 탐색이 외부에 있음 |
| **RAP** (Hao et al., 2023) | 2023 | MCTS + 세계 모델 | MCTS | 미검증 | 낮음 | 외부 세계 모델 필요 |
| **Stream of Search** (Gandhi et al., 2024) | 2024 | 탐색 과정 LM 학습 | 내재화 (선형 직렬화) | 중간 | 높음 | 트리 구조 없이 선형 직렬화 |
| **AlphaZero-like** (Feng et al., 2024) | 2024 | MCTS + LLM | MCTS | 미검증 | 낮음 | 외부 MCTS 의존 |
| **DeepSeek-R1** (DeepSeek, 2025) | 2025 | RL (GRPO) | 선형 롤아웃 | 중간 | 중간 | RL 필요, 선형 생성 |
| **ReJump** (Zeng et al., 2025) | 2025 | 트리 점프 표현 분석 | 트리 기반 | 미검증 | 중간 | 분석 도구 중심 |
| **TSLM** (Kim et al., 2026) | **2026** | **SFT (트리 직렬화)** | **내재화 (트리 구조)** | **높음** (OOD 91.5%) | **높음** | **이 논문** |

**Stream of Search (SoS)와의 핵심 차이**:
SoS도 탐색 과정을 언어 모델에 내재화하려 하지만, 탐색 궤적을 **선형 시퀀스**로 직렬화한다. TSLM은 명시적 **트리 구조**와 선택적 컨텍스트 분리를 통해 형제 노드 간 독립성을 보장하며, 이것이 OOD 일반화의 핵심이다.

## 결론

TSLM은 **"어떻게 생각하는가"가 아니라 "어떤 구조로 생각하는가"가 일반화를 결정한다**는 새로운 관점을 제시한다. 트리 구조의 내재화를 통해 강화학습 없이도 체계적 탐색 능력을 획득할 수 있음을 보였으며, 특히 OOD 일반화에서 탁월한 성능을 보인 점은 향후 LLM 추론 연구의 중요한 방향성을 제시한다. 그러나 계산 비용, 오픈 엔드 도메인 적용, 대규모 태스크 검증 등 해결해야 할 과제도 명확하다.
