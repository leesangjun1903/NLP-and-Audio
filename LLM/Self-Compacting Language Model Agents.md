## 1. 핵심 주장과 주요 기여 요약

### 🔑 핵심 주장

장기 에이전트 궤적(trajectory)에서 발생하는 **"컨텍스트 부패(Context Rot)"** — 즉, 오래되고 오류가 있는 중간 추론 결과들이 후속 생성을 오염시키는 현상 — 을 해결하기 위해, 모델 스스로 **"언제"**, **"어떻게"** 컨텍스트를 압축(compact)할지 결정하게 하는 **SELFCOMPACT** 스캐폴드를 제안합니다.

기존 방식들은 토큰 임계값(fixed-interval)에 도달하면 무조건 압축하는 방식이었으나, 이는 추론 중간에 필요한 정보를 삭제하는 문제를 일으켰습니다.

### 🏆 주요 기여 (3가지)

| 기여 | 내용 |
|------|------|
| **① 방법론 (SELFCOMPACT)** | 인라인 요약(summarization) 도구 + 경량 루브릭(rubric)을 결합한 새로운 스캐폴드 제안. **파인튜닝 불필요** |
| **② 실험적 검증** | 7개 오픈소스 모델, 6개 벤치마크에서 고정 간격 요약 방식을 동등하거나 초과하는 성능, **30~70% 낮은 토큰 비용** 달성 |
| **③ 메타인지 갭 발견** | 루브릭 없이 도구만 제공하면 모델이 압축 시점을 제대로 판단 못함을 발견 → "언제 압축할지"는 추론 시점에 루브릭으로 공급 가능 |

## 2. 상세 분석: 문제 → 방법 → 구조 → 성능 → 한계

### 2.1 해결하고자 하는 문제

#### 문제 1: 컨텍스트 부패 (Context Rot)

LM 에이전트가 장기 작업을 수행할수록 궤적에는 다음과 같은 **"쓰레기(junk)"** 가 쌓입니다:
- 초반에 잘못된 케이스 분리(case split)
- 이미 지나친 검색 결과
- 어디에도 이어지지 않는 중간 프로그램 후보

이러한 내용들은 단순히 메모리를 낭비하는 것이 아니라, **후속 생성을 능동적으로 오염**시킵니다. Hong et al.(2025)의 연구에 따르면, 오류 없이 처음부터 문제를 풀던 모델도 자신의 잘못된 추론을 다시 입력받으면 실패합니다.

#### 문제 2: 기존 전략의 구조적 결함

| 전략 | 방식 | 실패 원인 |
|------|------|-----------|
| **Reactive Compaction** | 컨텍스트가 토큰 한계에 도달할 때만 압축 | 이미 오염이 오래 지속된 후 압축 |
| **Periodic (Fixed-Interval) Compaction** | 매 $k$턴 또는 $k$토큰마다 압축 | 추론 **중간**에 압축 → 필요한 중간 결과 삭제 |

**핵심 문제**: 두 방식 모두 **"모델이 현재 무엇을 하고 있는가"를 전혀 모른 채** 압축을 실행합니다.

### 2.2 제안 방법 (수식 포함)

#### 기본 설정 (Setup)

자연어 프롬프트 $x$가 주어졌을 때, 언어 모델 $\pi(\cdot \mid x)$는 다음과 같이 연속(continuation)을 자기회귀적으로 생성합니다:

$$y = (y_1, y_2, \ldots)$$

각 토큰은 프롬프트와 선행 토큰 모두에 조건부입니다. 여기에 **요약 도구(summarization tool)** $\mathcal{S}$를 다음과 같이 정의합니다:

$$\mathcal{S} : (x, y_{1:t}) \rightarrow \tilde{y}$$

$\mathcal{S}$는 원본 프롬프트 $x$와 (부분적으로 생성된) 연속 $y_{1:t}$을 받아, 압축된 버전 $\tilde{y}$를 생성합니다. 생성은 새 컨텍스트 $(x, \tilde{y})$에서 재개됩니다. **요약기와 루브릭 판단기는 동일한 모델 $\pi$를 사용합니다.**

#### 압축 트리거 조건 (Triggering)

프로브 간격 $t \in \{N, 2N, 3N, \ldots\}$에서, 스캐폴드는 루브릭 프롬프트 $P_R$을 $(x, y_{1:t})$에 추가하고:

$$r_t \sim \pi(x, y_{1:t}, P_R), \quad r_t \in \{\text{COMPRESS}, \text{CONTINUE}\}$$

**COMPRESS** 판정이 나오면 요약기가 실행됩니다:

$$\tilde{y} \leftarrow \mathcal{S}(x, y_{1:t}) \quad \text{whenever } t \equiv 0 \pmod{N} \text{ and } r_t = \text{COMPRESS}$$

그렇지 않으면 생성이 $(x, y_{1:t})$에서 그대로 계속됩니다.

#### 비용 분석 (Cost Analysis)

$L = |y_{1:t}|$ (압축 전 궤적 길이), $\ell = |\tilde{y}|$ (요약 길이, 단위: 토큰)으로 정의하면:

- KV 캐시 재사용으로 $\mathcal{O}(L^2)$의 naive re-prefill 비용 회피
- 요약기의 실제 오버헤드: $\mathcal{O}(L\ell)$ (생성) + $\mathcal{O}(\ell^2)$ (새 prefix prefill)
- **비용 손익 분기점**: 압축이 이득인 조건 → $L/\ell > 10$ (실험적으로 검색 요약기는 $20\sim80\times$ 축소 달성)

**단일 요율(single-rate) 비용 모델**:

$$\text{COST}(q) = \frac{1}{10^6}\left(p_{\text{cache}} \cdot N_{\text{prompt}} + p_{\text{out}} \cdot N_{\text{out}}\right)$$

**세분화 이중 요율(two-rate) 비용 모델**:

$$\text{COST}(q) = \frac{1}{10^6}\left(p_{\text{in}} \cdot N_{\text{prefill}} + p_{\text{cache}} \cdot N_{\text{cached}} + p_{\text{out}} \cdot N_{\text{out}}\right)$$

### 2.3 모델 구조 (SELFCOMPACT 알고리즘)

논문에 제시된 알고리즘 1을 재구성하면:

```
입력: 프롬프트 x, 모델 π, 프로브 간격 N, 스텝 예산 T, 루브릭 프롬프트 P_R, 요약 프롬프트 P_S

C ← x  // 전체 컨텍스트 및 KV 캐시

for t = 1, ..., T do:
    yt ~ π(·|C);  C ← C ∘ yt
    if yt = final answer: return yt
    
    if t mod N = 0:
        C ← C ∘ P_R            // 루브릭 프로브 (KV prefix 재사용)
        rt ~ π(·|C), rt ∈ {COMPRESS, CONTINUE}
        C ← C ∘ rt
        
        if rt = COMPRESS:
            C ← C ∘ P_S        // KV prefix 여전히 재사용
            ỹ ~ π(·|C)          // 궤적 요약
            C ← x ∘ ỹ          // 하드 리셋: 요약에서 재개
        else:
            pop(C, rt); pop(C, P_R)  // 원복: 궤적 불변

return yT
```

#### 두 가지 핵심 요소의 상호보완성

```
┌─────────────────┐    ┌─────────────────────────────┐
│  요약 도구(S)    │    │  경량 루브릭(Rubric)         │
│                 │    │                             │
│ 도구만 있을 때: │    │ 루브릭만 있을 때:           │
│ • 일부 모델:    │    │ • 행동 불가                 │
│   도움 안 되는  │ +  │ • 판단만 하고 실행 못 함    │
│   시점에 호출   │    │                             │
│ • 일부 모델:    │    │ 함께 사용 시:               │
│   전혀 미호출   │    │ • 효과적인 적응형 압축       │
└─────────────────┘    └─────────────────────────────┘
```

#### 루브릭의 구체적 조건

**수학 루브릭** (COMPRESS iff $Q1=Y \lor (Q2=Y \land Q3=Y)$ ):
- **Q1 ANSWER**: 최신 라운드에 구체적인 최종 답변이 있는가?
- **Q2 STUCK**: 최근 2라운드에 비자명한 새 사실이 없는가?
- **Q3 HAS-NEXT**: 정확한 다음 단계를 쓸 수 있는가?

**검색 루브릭** (COMPRESS iff $C1=Y \land C2=Y \land C3=Y \land N1=N$):
- **C1 CLOSED-UNIT**: 궤적이 닫힌 단위(completed sub-task)에 도달했는가?
- **C2 SUMMARIZABLE**: 3~5개의 인용 가능한 핵심 사실로 축약 가능한가?
- **C3 PROGRESS**: 마지막 압축 이후 새로운 구체적 사실을 얻었는가?
- **N1 STUCK**: 최근 4번 검색 중 3번 이상이 새 정보 없이 중복이었는가? (N이어야 COMPRESS)

### 2.5 한계

#### ① 모델 역량 격차 (Model Capability Gap)
- **오픈소스 모델만 평가** (GPT-5.5, Claude Opus 4.7, Gemini 3 Pro 등 프론티어 모델 미평가)
- 프론티어 모델은 루브릭 없이도 컨텍스트 부패를 탐지할 가능성 있음 (Lindsey, 2025의 메타인지 연구 참조)

#### ② 강화학습 미적용 (No Reinforcement Learning)
- 훈련 없는 개입(training-free intervention)으로만 범위 제한
- Kontonis et al.(2026), Zhu et al.(2026b)는 RL로 모델이 언제/무엇을 압축할지 학습 가능함을 보임
- 루브릭이 RL의 행동 목표(behavioral target)로 활용될 수 있는 자연스러운 확장 가능성 언급

#### ③ 루브릭의 태스크 특이성
- 수학 루브릭과 검색 루브릭이 별도로 설계됨 → 새 도메인에 대한 일반화를 위해 별도의 루브릭 설계 필요

## 3. 모델의 일반화 성능 향상 가능성

### 3.1 일반화의 근거: 학습 없는 메타인지 공급

SELFCOMPACT의 핵심 발견은 다음과 같습니다:

> **"언제 압축할지(when to compact)"는 훈련 없이 추론 시점에 루브릭으로 공급될 수 있는 메타인지적 역량이다.**

이것이 일반화에 중요한 이유:
- 모델 가중치를 수정하지 않으므로, 새로운 모델에 적용 시 **재훈련 불필요**
- 7개 모델 (Qwen 계열 4개 + GLM/MiniMax/MiMo 3개)에서 일관된 효과 → **아키텍처 독립성** 시사

### 3.2 어려운 문제에서의 일반화 우위

논문의 난이도별 분석(Figure 3)에서:

- **쉬운 문제**: SELFCOMPACT ≈ Fixed-interval ≈ Baseline (차이 없음)
- **어려운 문제(상위 2 분위)**: SELFCOMPACT가 Threshold 대비 **5~20pp 일관되게 우세**

이는 SELFCOMPACT가 **컨텍스트가 크게 축적되는 복잡한 문제**에서 일반화가 더 잘 됨을 시사합니다.

### 3.3 조기 압축의 일반화 이점

Figure 2에서 SELFCOMPACT의 요약 발화 시점 분포가 고정 임계값(30%) **왼쪽으로 치우침**:
- 고정 임계값은 항상 30%에서 발화 → 모델이 이미 하위 질문을 해결한 후에도 계속 쌓임
- SELFCOMPACT는 하위 작업 완료 시점에 발화 → **불필요한 맥락 없이 다음 하위 작업으로 이동**

이것이 새로운 태스크에서의 일반화 향상 메커니즘:
$$\text{압축 시점의 정확성} \propto \text{후속 추론의 오염 감소} \propto \text{새 하위 태스크에서의 일반화 향상}$$

### 3.4 루브릭의 이식 가능성과 한계

**이식 가능성**:
- 루브릭은 자연어로 작성된 조건 목록이므로, **도메인 변경 시 루브릭 텍스트만 수정**하면 됨
- 외부 검증자, 파인튜닝, 별도 모델 불필요

**한계**:
- 수학 루브릭과 검색 루브릭이 별도 설계 → 완전한 도메인 일반화를 위한 루브릭 자동 생성 연구 필요
- 루브릭의 품질이 압축 적시성에 직접 영향 → 루브릭 설계 방법론의 체계화 필요

## 4. 2020년 이후 관련 최신 연구 비교 분석

아래 비교는 논문 원문에 인용된 문헌들을 기준으로 정리하였습니다.

### 4.1 컨텍스트 압축 방법론 비교

| 연구 | 방법 | 트리거 방식 | 훈련 필요 여부 | SELFCOMPACT와의 차이 |
|------|------|-------------|----------------|---------------------|
| **ReasoningCache** (Wu et al., 2026a) | 단기 RL로 반복 압축-재개 | 고정 토큰 임계값 | 필요 (RL) | 트리거가 구조 무관 |
| **InftyThink** (Yan et al., 2026) | 무한 컨텍스트를 위한 반복 요약 | 고정 토큰 임계값 | 필요 (SFT) | 트리거가 구조 무관 |
| **FoldAct** (Shao et al., 2025) | 검색 에이전트 컨텍스트 폴딩 | 고정 간격 | 필요 | 고정 간격 의존 |
| **Context-Folding** (Sun et al., 2025) | LLM 에이전트 컨텍스트 폴딩 | 고정 간격 | 필요 | 고정 간격 의존 |
| **Re-Trac** (Zhu et al., 2026a) | 외부 프론티어 모델로 요약 | 고정 검색 궤적 후 | **불필요** | 외부 모델 의존, 고정 간격 |
| **Memento** (Kontonis et al., 2026) | RL로 컨텍스트 관리 학습 | 학습된 정책 | 필요 (RL) | 훈련 없는 접근 불가 |
| **Accordion-Thinking** (Yang et al., 2026) | 단계별 자기규제 요약 | 단계 경계 | 필요 (SFT) | 구조적이지만 훈련 필요 |
| **SELFCOMPACT** (본 논문) | 루브릭 게이팅 + 인라인 요약 도구 | **적응형 (루브릭)** | **불필요** | 최초의 루브릭 기반 적응형 트리거 |

### 4.2 KV 캐시 축소 접근법과의 비교

| 연구 | 방법 | 해석 가능성 | 목적 |
|------|------|-------------|------|
| **SnapKV** (Li et al., 2024) | 중요도 기반 KV 캐시 퇴출 | 낮음 (블랙박스) | 메모리 효율 |
| **Attention Sinks** (Xiao et al., 2023) | 최근성 기반 캐시 유지 | 낮음 | 스트리밍 효율 |
| **KeyDiff** (Park et al., 2025) | 키 유사성 기반 KV 퇴출 | 낮음 | 자원 제한 환경 |
| **SELFCOMPACT** | 자연어 요약으로 컨텍스트 대체 | **높음 (인간 가독)** | 정확도 + 비용 |

핵심 차이: KV 캐시 접근법은 표현의 해석 가능성을 부차적으로 취급하지만, SELFCOMPACT는 **인간 가독한 자연어 요약**을 유지합니다.

### 4.3 LangChain 동시 연구와의 비교

논문은 2026년 3월 LangChain이 동시에 요약 도구를 도입했음을 언급하지만, LangChain의 접근법은 **루브릭 기반 판단 기준을 도입하지 않았음**을 명시합니다. SELFCOMPACT가 최초로 루브릭 기반 적응형 타이밍을 훈련 없이 평가한 연구입니다.

## 5. 앞으로의 연구에 미치는 영향과 고려 사항

### 5.1 연구에 미치는 영향

#### ① 메타인지의 스캐폴드 공급 가능성

SELFCOMPACT는 **"메타인지 역량(meta-cognitive capability)을 가중치에 구울 필요 없이 추론 시점에 공급할 수 있다"** 는 원칙을 확립합니다. 이는:
- 자기 모니터링(self-monitoring)
- 불확실성 인식(uncertainty awareness)
- 전략 전환(strategy switching)

등 다른 메타인지적 역량에도 동일한 원리를 적용할 수 있는 연구 방향을 제시합니다.

#### ② 장기 에이전트 시스템의 컨텍스트 관리 표준

오라클 분석이 보여주듯, 적응형 압축 정책에는 **52.9%** 에 달하는 상당한 헤드룸이 있습니다. 이는:
- 더 정교한 압축 타이밍 학습 연구를 촉진
- RL을 통한 루브릭 내재화(distillation)로의 자연스러운 연구 방향 제시
- 컨텍스트 관리가 장기 에이전트의 핵심 평가 축으로 자리 잡을 것을 시사

#### ③ 비용-성능 트레이드오프 재정의

30~70% 비용 절감 + 성능 향상의 동시 달성은 기존의 **"더 많은 토큰 = 더 좋은 성능"** 패러다임에 도전하며, **토큰 예산의 효율적 배분**이 중요한 연구 문제임을 보여줍니다.

### 5.2 앞으로 연구 시 고려할 점

#### 🔬 방법론적 확장

1. **루브릭의 자동 생성/최적화**
   - 현재 루브릭은 수동 설계 → 새 도메인에 적용 시 수동 설계 부담
   - 루브릭 자체를 LLM으로 생성하거나 RL로 최적화하는 연구 필요

2. **강화학습과의 결합**
   - 루브릭이 정의하는 압축 타이밍을 RL의 보상 신호로 활용 (Kontonis et al., 2026 방향)
   - 수식으로 표현하면:
   $$r_{\text{RL}}(s_t) = \mathbb{1}[\text{rubric}(s_t) = \text{COMPRESS}] \cdot \Delta_{\text{accuracy}}$$
   - 루브릭이 제공하는 "언제"와 RL이 제공하는 "어떻게"의 결합으로 시너지 기대

3. **다중 압축 수준(Multi-level Compaction)**
   - 현재: 전체 궤적을 단일 요약으로 대체
   - 개선 가능: 하위 작업 수준, 검색 세션 수준, 전체 수준의 계층적 압축

4. **요약 품질 평가 지표 개발**
   - 현재 성능은 최종 정확도로만 측정
   - 요약의 정보 보존율, 핵심 사실 유지율을 직접 측정하는 지표 개발 필요

#### 📊 평가 확장

5. **프론티어 모델 평가**
   - GPT-5.5, Claude Opus 4.7, Gemini 3 Pro 등에 대한 SELFCOMPACT 적용 가능성 검증
   - 더 강한 메타인지를 가진 모델에서 루브릭의 역할 변화 분석

6. **다양한 도메인 확장**
   - 코딩 에이전트 (SWE-bench 등)
   - 멀티모달 에이전트
   - 장기 대화 시스템

7. **루브릭 조건의 민감도 분석**
   - 루브릭 조건의 변화가 성능에 미치는 영향 체계적 분석
   - 최적 루브릭 조건 수, 증거 요구 수준 등의 하이퍼파라미터 연구

#### ⚙️ 시스템적 고려

8. **KV 캐시 재사용 최적화**
   - 현재 KV 캐시 재사용이 비용 절감의 핵심
   - 다양한 추론 엔진(vLLM 외)에서의 구현 및 최적화 연구

9. **압축 불가역성(Irreversibility) 완화**
   - 현재 압축은 비가역적 → 잘못된 요약 시 복구 불가
   - 요약 전 원본의 일부를 유지하거나 체크포인트를 활용하는 설계 연구

10. **루브릭의 공정성(Fairness)과 편향**
    - 루브릭이 특정 유형의 추론(예: 순차적 추론 > 병렬 탐색)에 유리하게 작용할 가능성 분석

## 📝 종합 정리

SELFCOMPACT는 LM 에이전트의 장기 추론에서 **컨텍스트 관리를 모델 스스로의 메타인지적 결정으로 격상**시킨 최초의 훈련-자유 접근법입니다. 핵심 수식:

$$\tilde{y} \leftarrow \mathcal{S}(x, y_{1:t}) \quad \text{iff} \quad t \equiv 0 \pmod{N} \wedge r_t = \text{COMPRESS}$$

를 통해, "얼마나 많은 토큰이 지났는가"가 아닌 **"추론 구조상 압축이 안전한 시점인가"** 를 판단하여, 6개 벤치마크, 7개 모델에서 성능 향상과 비용 절감을 동시에 달성했습니다. 앞으로의 연구는 이 루브릭의 자동화, RL과의 결합, 그리고 더 다양한 도메인으로의 확장에 집중될 것으로 예상됩니다.
