## 1. 핵심 주장 및 주요 기여 요약

Microsoft AI 팀이 이 논문에서 반복적으로 강조하는 핵심 메시지는, 그들이 단순히 **모델을 만들려 한 것이 아니라 "Hill-Climbing Machine"을 구축하려 했으며, 모델은 그 산출물(product)**이라는 점입니다.

이 프로세스는 사전학습(pre-training) 모델링 결정을 위한 스케일링 중심 프레임워크, 그리고 장기적이고 로그-선형적인 성능 향상을 유지하는 강건한 강화학습(RL) 레시피 및 인프라를 포함합니다.

이 프로세스를 통해 개발된 첫 번째 모델인 MAI-Thinking-1은 **35B 활성 파라미터 / 1T 총 파라미터 MoE(Mixture-of-Experts)** 모델로, STEM 추론 및 코딩 과제에서 동급 최강 모델들 중 하나이며, AIME 2025에서 97.0%, SWE-Bench Pro에서 52.8%, LiveCodeBench v6에서 87.7%를 기록했습니다. 이 모델은 완전히 처음부터(from-scratch) 학습되었으며, 제3자 모델로부터의 지식 증류(distillation) 없이 순수하게 자체 데이터로만 학습되었습니다.

### 세 가지 핵심 철학적 기둥

Hill-Climbing Machine의 개발 파이프라인은 세 가지 기둥으로 구성됩니다: **① 학습된 역량(distillation 금지)**, **② 정제된/라이선스된 사전학습 데이터**, **③ 엔드-투-엔드 자체 구축 훈련 인프라**.

- **첫째, 역량은 학습되어야지 상속되어선 안 됩니다.** 상속된 지능(증류)은 실제 세계 사용에 필수적인 조종 가능성(steerability)이 부족하며, 모방자는 교사의 설계 선택에 근본적으로 묶여 새로운 상황에 적응하기 어렵습니다.
- **둘째, 깨끗한 데이터.** MAI-Thinking-1은 AI 생성 콘텐츠를 사전학습에서 제외한 깨끗하고 적절히 라이선스된 데이터로 학습되었습니다. 이는 품질, 출처 명확성, 통제 가능성에 중요하며, 모델을 형성한 요소들을 파악하지 못하면 그 행동을 완전히 이해하거나 신뢰성 있게 개선할 수 없기 때문입니다.
- **셋째, 전 스택에 걸친 자급자족(self-sufficiency).** MSFT 자체 가속기(Maia 200)와의 모델 공동 설계부터 강화학습 프레임워크에 이르기까지, 자체 훈련 인프라에 집중하였으며, 이는 시스템을 완전히 최적화하고 요구에 맞게 형성할 수 있도록 Hill-Climbing Machine 구축의 핵심 부분입니다.

## 2. 해결하고자 하는 문제, 제안 방법, 모델 구조, 성능 향상 및 한계

### 2-1. 해결하고자 하는 문제

이 논문이 다루는 핵심 문제는 크게 세 가지입니다.

**① 지속적·반복적 성능 향상 체계 부재**: 기존 AI 개발은 단일 모델 출시에 집중하여, 다음 사이클에서 일관되게 더 나은 모델을 만들 수 있는 **반복 가능한(repeatable) 개선 루프**가 없었습니다.

**② 제3자 의존성 및 불투명한 데이터 문제**: 프론티어 모델로 가는 지름길은 없으며, 타 연구소로부터의 증류나 미인가/불투명한 데이터에 의존하는 것은 신뢰성과 통제 가능성을 저해합니다.

**③ 안전성-유용성 트레이드오프**: MAI-Thinking-1 개발의 핵심 기술적 과제는, 사용자 요청을 거부하도록 요구하는 **안전성**과 요청을 이행하도록 요구하는 **유용성** 사이의 균형을 맞추는 것이었으며, 안전한 모델은 안전 정책과 기준을 준수하면서도 유용한 응답을 제공해야 합니다.

---

### 2-2. 제안하는 방법

#### (A) 사전학습 (Pre-training)

MAI-Thinking-1은 웹 데이터, 공개 GitHub 코드, 도서, 학술 논문, 뉴스, 다국어 텍스트, 도메인 특화 자료 등 **공개적으로 이용 가능한 데이터와 라이선스된 인간 생성 데이터 혼합으로부터 추출된 30T 토큰**으로 사전학습 되었습니다. 이 소스들은 모두 자체 처리되었으며, 언어 모델이 생성한 합성 데이터는 사전학습에 사용하지 않았습니다.

모델은 **8,192개의 GB200 GPU**를 사용하여 학습되었으며, **256K 컨텍스트 윈도우**를 지원합니다.

#### (B) 강화학습 (RL) 기반 Post-training: 전문가 분리-병합 전략

RL 학습은 세 개의 전문가 모델로 분리하여 진행됩니다. 하나는 STEM 및 경쟁 코드를 담당하고, 하나는 에이전틱 코딩 및 도구 사용을 담당하며, 하나는 유용성 및 안전성을 담당합니다. 이후 지도 학습(supervised pass)을 통해 세 모델을 하나로 증류한 뒤, 최종 RL 클라이밍을 통해 MAI-Thinking-1이 완성됩니다. 이렇게 분리 후 병합하는 이유는, 세 도메인이 서로 다른 보상을 요구하므로 세 팀이 서로 간섭 없이 병렬로 학습할 수 있기 때문입니다.

#### (C) 안전성의 RL 통합

유용성과 안전성 훈련을 RL 클라이밍 과정 내에 통합하여 내부 안전성 벤치마크를 기준으로 삼으며, 책임 있는 배포의 일환으로 모델 개발 전반에 걸쳐 지속적인 레드팀(red-teaming)을 수행하여 출시 전 취약점을 발견하고 해결합니다.

안전하지 않은 순응과 불필요한 거부 모두를 역량과 동일한 강화학습 루프 내에서 결함으로 처리합니다.

#### (D) 스케일링 법칙 활용

Hill-Climbing Machine의 핵심 방법론은 **스케일링-집중(scaling-focused)** 프레임워크입니다. 이는 다음의 수식으로 일반화할 수 있습니다.

사전학습 스케일링 법칙 (Chinchilla-style):

$$L(N, D) = A \cdot N^{-\alpha} + B \cdot D^{-\beta} + L_0$$

여기서:
- $L$: 검증 손실(validation loss)
- $N$: 모델 파라미터 수
- $D$: 학습 토큰 수
- $A, B, \alpha, \beta$: 경험적으로 추정되는 스케일링 계수
- $L_0$: 데이터 분포에 의한 하한 손실

RL 목적함수 (GRPO/PPO 계열):

$$\mathcal{J}(\theta) = \mathbb{E}_{\tau \sim \pi_\theta}\left[\sum_{t=1}^{T} \min\left(\rho_t \hat{A}_t,\ \text{clip}(\rho_t, 1-\epsilon, 1+\epsilon)\hat{A}_t\right)\right] - \beta \cdot \mathbb{D}_{\text{KL}}[\pi_\theta \| \pi_{\text{ref}}]$$

여기서:
- $\rho_t = \frac{\pi_\theta(a_t|s_t)}{\pi_{\text{old}}(a_t|s_t)}$: 중요도 가중치(importance weight)
- $\hat{A}_t$: 어드밴티지 추정치(advantage estimate)
- $\epsilon$: 클리핑 파라미터
- $\beta$: KL 정규화 계수
- $\pi_{\text{ref}}$: 참조 정책(reference policy)

> ⚠️ **주의**: 위 수식은 논문이 사용하는 일반적인 RL 프레임워크를 기반으로 한 것이며, 공식 기술 보고서 내 정확한 수식 표기는 원문 PDF에서 확인을 권장합니다.

### 2-3. 모델 구조

MAI-Thinking-1은 **35B 활성 파라미터, 약 1T 총 파라미터**를 갖는 희소 MoE(Mixture of Experts) 모델로, 훨씬 더 큰 모델보다 작은 추론 풋프린트를 가집니다.

**피드포워드 및 MoE 레이어 인터리빙 설계:**

각 블록의 피드포워드 레이어에서 **MoE 레이어와 밀집 피드포워드 네트워크(FFN)를 교대(interleaved)** 배치합니다. 이 방식(고희소성 레이어 + 제로희소성 레이어 페어링)은 균형 잡힌 희소성 할당과 비교 가능한 스케일링 성능을 보이면서, iso-active 및 iso-total 파라미터 설정 모두에서 벽시계 시간(wall-clock time) 효율이 더 높습니다. 또한 매 레이어 MoE 배치는 공유 전문가(shared experts)에 훨씬 더 크게 의존하지만, 인터리빙 배치에 공유 전문가를 추가해도 이점이 거의 없습니다.

첫 번째 피드포워드 레이어는 밀집(dense) 레이어이며, **SwiGLU** 활성화 함수를 사용합니다.

주요 구조 요약:

| 항목 | 내용 |
|------|------|
| 아키텍처 | Sparse MoE (Interleaved MoE + Dense FFN) |
| 활성 파라미터 | 35B |
| 총 파라미터 | ~1T |
| 컨텍스트 윈도우 | 256K |
| 사전학습 토큰 | 30T |
| 학습 하드웨어 | 8,192× GB200 GPU |
| 활성화 함수 | SwiGLU |
| 어텐션 최적화 | FlashAttention-4 + Ulysses-style context parallelism |
| 커스텀 칩 | Maia 200 (1.4× 효율 향상) |

자체 Maia 200 실리콘과의 공동 설계를 통해 이미 **1.4배의 효율 향상**을 확인하였습니다.

### 2-4. 성능 향상

MAI-Thinking-1은 AIME 2025에서 **97.0%**, AIME 2026에서 **94.5%**, LiveCodeBench v6에서 **87.7%**를 달성했습니다.

MAI-Thinking-1(35B 활성, ~1T 총 파라미터, MoE)은 훨씬 더 큰 모델보다 작은 추론 풋프린트를 가지면서도 SWE-Bench Pro에서 Claude Opus 4.6과 경쟁적이며, Surge의 독립적 인간 평가자들의 블라인드 비교에서 단일 및 멀티턴 태스크 전반에 걸쳐 전체 품질에서 Sonnet 4.6보다 선호되었습니다.

### 2-5. 한계점

분리(specialists) 후 병합(merge) 전략의 위험은 **통합 과정이 전문가들의 피크 성능을 평탄화(flatten)할 수 있다**는 것입니다. 세 분야에서 모두 좋은 단일 모델은 세 개의 단일 도메인 모델이 유지하는 엣지를 포기하게 되며, 최종 RL 클라이밍이 이를 일부 회복하려 하지만, 이것이 실제로 달성되는지는 벤치마크만으로는 답할 수 없는 열린 질문입니다.

추가적으로:
- 사전학습 시 합성 데이터를 전혀 사용하지 않는 원칙은 데이터 품질과 신뢰성에서는 강점이지만, 도메인 특화 데이터 부족 상황에서의 대응력을 제약할 수 있습니다.
- 추론 중심 강화학습 post-training(예: 명시적 chain-of-thought를 위한 RL)은 **안전성 정렬을 저하시킬 수 있으며**, 이는 jailbreak 공격에 대한 취약성을 높일 수 있다는 점이 관련 연구에서 지속적으로 보고되고 있습니다.

## 3. 모델의 일반화 성능 향상 가능성

이 논문에서 일반화 성능 향상과 관련된 핵심 전략은 다음과 같습니다.

### 3-1. "학습된 역량"의 일반화 우위

역량은 학습되어야 하며, 상속된(증류된) 지능은 실제 세계 사용에 필수적인 조종 가능성이 부족합니다. 모방자는 교사의 설계 선택에 근본적으로 묶여 **새로운 상황에 적응하기 어렵습니다**. 즉, 증류 없이 처음부터 학습된 모델이 분포 외(out-of-distribution) 상황에서 더 나은 일반화를 보일 잠재력을 갖습니다.

### 3-2. RL의 일반화 잠재력

관련 선행 연구(arXiv:2509.25300)에 따르면, 고정된 컴퓨팅 예산 하에서 적은 단계로 학습된 **더 큰 모델이 더 많은 단계로 학습된 더 작은 모델을 일관되게 능가**합니다. 고정된 학습 데이터량에서는 더 큰 모델이 더 우수한 샘플 효율성을 달성합니다. MAI-Thinking-1은 1T 파라미터 규모로 이 이점을 누립니다.

이러한 스케일링 특성은 베이스 모델과 명령어 튜닝 모델 모두에서 강건하게 나타납니다.

### 3-3. 도메인 전문화 후 통합을 통한 범용성

STEM 추론, 에이전틱 코딩, 유용성/안전성의 세 전문가 모델을 각각 독립적으로 학습한 후 하나로 통합하는 구조는, 각 도메인의 특화된 추론 능력을 일반 모델에 내재화함으로써 폭넓은 일반화를 추구합니다.

### 3-4. 반복적 RL 클라이밍과 log-linear 성능 개선

깨끗한 데이터 파이프라인, 특화된 학습 인프라, 강화학습 환경을 긴밀히 연결함으로써, 컴퓨팅이 확장될수록 더 높은 역량을 향해 꾸준히 "등반"하도록 설계된 **최적화 루프**가 구성됩니다.

실세계 환경에서의 강화학습을 통해 AI는 처음으로 주어진 워크플로우의 구체적인 사항에 완전히 적응할 수 있게 됩니다. 이는 단순한 벤치마크 성능을 넘어서는 **실제 환경 일반화**를 의미합니다.

### 3-5. 합성 데이터 배제의 일반화 함의

사전학습에서 언어 모델이 생성한 합성 데이터를 사용하지 않고 AI 생성 콘텐츠를 제거하는 노력을 기울였습니다. 이는 모델 붕괴(model collapse) 위험을 줄이고, 인간 생성 데이터의 다양성과 자연스러운 분포를 보존하여 **장기적 일반화 성능**을 높이는 데 기여합니다.

## 4. 앞으로의 연구에 미치는 영향과 고려할 점

### 4-1. 향후 연구에 미치는 영향

**① "시스템으로서의 AI 개발" 패러다임 전환**

단일 모델이 아닌, **Hill-Climbing Machine**: 더 나은 데이터, 더 강한 보상, 더 유능한 환경, 더 많은 컴퓨팅을 흡수할 수 있는 반복 가능한 시스템을 구축하는 것이 핵심 패러다임입니다. 이는 AI 연구에서 "모델 출시"가 아닌 "개선 루프 설계"가 경쟁의 핵심이 됨을 시사합니다.

**② 증류 없는 처음부터의 학습의 가능성 재조명**

모든 모델이 제로 증류로 공유 기반 위에서 구축되며, 동일한 데이터 원칙, 인프라, 평가 프레임워크를 공유합니다. 이는 "증류 기반 접근법만이 효율적"이라는 업계의 암묵적 가정에 도전하며, **자체 개발 파이프라인의 경쟁력**을 실증합니다.

**③ 안전성을 RL 루프에 통합하는 접근법의 확산**

안전성은 역량에 사용하는 것과 동일한 강화학습 인프라로 학습되어, 안전성 보상이 동일한 hill-climbing 루프의 일부가 됩니다. 이는 기존에 사후 과정으로 취급되던 안전성을 **학습의 핵심 목적함수에 내재화**하는 방향으로 연구 방향을 이끌 것입니다.

**④ 하드웨어-모델 공동 설계의 중요성 부각**

Maia 200 실리콘과의 공동 설계를 통해 이미 1.4배의 효율 향상을 보이고 있어, 칩과 모델의 공동 최적화가 단순한 소프트웨어 최적화를 뛰어넘는 핵심 경쟁력임을 보여줍니다.

### 4-2. 향후 연구 시 고려할 점

| 고려 항목 | 설명 |
|-----------|------|
| **전문가 분리-통합 전략의 최적화** | 통합 과정에서 전문가들의 피크 성능이 평탄화될 위험이 있으므로, 전문가 모델 병합 시 성능 손실을 최소화하는 방법론 연구가 필요합니다. |
| **RL과 안전성의 상충 문제** | 추론 중심 post-training은 안전성 정렬을 저하시킬 수 있으며, 이는 동일한 레시피가 추론 역량은 높이면서 안전성을 현저히 약화시킬 수 있음을 의미합니다. 역량과 안전성을 동시에 향상하는 보상 설계 연구가 요구됩니다. |
| **합성 데이터 없는 학습의 한계** | 사전학습에서 AI 생성 콘텐츠를 배제하는 원칙은 데이터 확장성에 제약을 줄 수 있으며, 고품질 합성 데이터의 활용 가능성을 탐구할 필요가 있습니다. |
| **RL 스케일링 법칙의 정립** | RL post-training 하에서의 스케일링 특성은 아직 충분히 탐구되지 않았습니다. 안정적이고 예측 가능한 RL 스케일링 법칙을 정립하는 연구가 필요합니다. |
| **일반화 vs. 전문화 균형** | 세 도메인이 서로 다른 보상을 요구하는 상황에서의 병렬 학습 후 통합은 전문 성능의 손실 가능성을 내포합니다. 멀티도메인 일반화를 유지하면서 전문 성능을 극대화하는 방법이 중요한 연구 주제입니다. |
| **로그-선형 성능 향상의 지속 가능성** | 프론티어 모델 훈련에 사용된 컴퓨팅은 이미 1조 배 증가하였고, 향후 3년간 추가로 1,000배 증가가 예상됩니다. 무한한 컴퓨팅 확장 가정 하에서 현재의 log-linear 개선이 유지될 수 있는지에 대한 검증이 필요합니다. |

## 5. 2020년 이후 관련 최신 연구 비교 분석

| 모델/연구 | 연도 | 방법론 | 비교 포인트 |
|-----------|------|--------|------------|
| **GPT-4** | 2023 | 대규모 사전학습 + RLHF | 증류 활용, 데이터 출처 불투명 |
| **DeepSeek-R1** (arXiv:2501.12948) | 2025 | RL로 추론 역량 강화(GRPO) | 오픈소스, RL 중심, 증류 모델 변형 포함 |
| **DeepSeek-R1**은 DeepSeek-V3-Base 위에서 강화학습으로 학습된 추론 중심 모델 패밀리입니다. | | | MAI-Thinking-1은 증류 없이 처음부터 학습하는 점에서 차별화됩니다. |
| **Scaling Behaviors of LLM RL Post-Training** (arXiv:2509.25300) | 2025 | RL post-training 스케일링 법칙 경험적 연구 | 54개 실험을 통해 더 큰 모델이 적은 단계로 학습 시 일관되게 우수함을 보임 — MAI-Thinking-1의 대규모 MoE 선택을 지지 |
| **Safety Recovery in Reasoning Models** (arXiv:2602.11096) | 2026 | RL 추론 학습 후 안전성 저하 분석 | 추론 역량을 높이는 레시피가 안전성을 현저히 약화시킬 수 있음 — MAI-Thinking-1의 RL 루프 내 안전성 통합 접근이 이 문제를 직접 겨냥 |
| **Inference-Time Scaling for Generalist Reward Modeling** (ICLR 2026) | 2026 | 추론 시간 스케일링으로 RM 개선 | 다양한 도메인에서 정확한 보상 신호 획득이 핵심 과제 — MAI-Thinking-1의 도메인별 전문가 RL과 상보적 관계 |
| **MAI-Thinking-1** (Microsoft AI, 2026) | 2026 | 처음부터 학습 + MoE + 분리-통합 RL + 안전성 내재화 | 증류 금지, 자체 데이터, 자체 하드웨어, RL-safety 통합 |

**핵심 차별점 요약**: MAI-Thinking-1이 DeepSeek-R1 등 동시대 강력한 추론 모델들과 다른 가장 큰 특징은 **① 증류 완전 배제**, **② 안전성을 RL 목적함수에 내재화**, **③ 자체 하드웨어(Maia 200) 공동 설계**, **④ 반복 가능한 시스템(Hill-Climbing Machine)으로서의 설계 철학**입니다.
