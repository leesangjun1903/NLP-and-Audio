## 1. 핵심 주장 및 주요 기여 요약

### 1.1 핵심 주장

머신러닝 모델의 동작을 이해하고 설명하는 것은 투명하고 신뢰할 수 있는 AI 시스템 구축에 필수적이다. 이를 위해 DEXTER는 **Diffusion 모델**과 **대형 언어 모델(LLM)**을 활용하여 시각적 분류기에 대한 전역적(global)·텍스트적 설명을 생성하는 **데이터 불필요(data-free) 프레임워크**다.

기존 연구와 달리, DEXTER는 학습 데이터나 정답 레이블 없이도 분류기의 의사결정 과정에 대한 자연어 설명을 제공할 수 있다.

### 1.2 주요 기여

DEXTER의 핵심 기여는 다음과 같다: **전역적 설명(Global Explanations)** — 로컬 어트리뷰션을 넘어, 핵심 특징, 편향, 패턴을 포함한 모델 의사결정에 대한 고수준 이해를 제공한다.

이미지 생성과 텍스트 추론을 결합함으로써, DEXTER는 시각적 설명의 해석 가능성 문제를 극복하고, 학습 데이터나 정답 레이블 없이 허위 상관관계 및 편향을 포함한 전역적 모델 행동 이해를 제공한다.

DEXTER는 세 가지 태스크 — **활성화 최대화(Activation Maximization)**, **슬라이스 탐색 및 편향 제거(Slice Discovery & Debiasing)**, **편향 설명(Bias Explanation)** — 에 걸쳐 유연성을 입증하며, 각각 시각 분류기의 내부 메커니즘을 밝힌다.

정량적·정성적 평가(사용자 연구 포함)를 통해 DEXTER가 정확하고 해석 가능한 결과물을 생성함을 확인했다.

## 2. 상세 분석

### 2.1 해결하고자 하는 문제

GradCAM, Integrated Gradients 등 기존 모델 설명 가능성 기법들은 **로컬 설명(local explanation)** 만을 제공하며 데이터를 필요로 한다. 이 방법들은 특정 픽셀이나 영역에 중요도를 귀속시켜 개별 예측을 분석하는 데 집중하지만, **모델 행동의 전역적 이해**를 제공하지 못한다.

반면, 활성화 최대화(Activation Maximization) 기법은 신경망이 학습한 특징을 전역적으로 시각화하는 데 기여했지만, 생성된 이미지가 추상적이어서 해석하기 어려운 경우가 많다. 기존 방법들은 전반적으로 예측 뒤의 **전역적 추론 패턴과 편향**을 포착하는 데 부족하다.

### 2.2 제안하는 방법 및 수식

#### 2.2.1 전체 파이프라인

DEXTER의 프레임워크는 세 가지 핵심 구성요소를 통합한다: **프롬프트 최적화를 위한 텍스트 파이프라인**, **이미지 생성 프로세스를 위한 비전 파이프라인**, **VLM을 사용한 추론 모듈**.

#### 2.2.2 소프트 프롬프트 최적화 (Text Pipeline)

DEXTER는 BERT 모델이 사전 정의된 문장에서 마스킹된 토큰을 채우도록 **소프트 프롬프트(soft prompt)를 최적화**하는 것으로 시작한다. 결과 프롬프트는 스테이블 디퓨전(Stable Diffusion) 프로세스를 유도하여 주어진 시각적 분류기에서 **대상 뉴런(예: 분류 헤드)의 활성화를 최대화**하는 이미지를 생성한다.

프롬프트의 구조를 수식으로 표현하면:

$$t = [t_{\text{fixed}}; m_1, m_2, \ldots, m_N]$$

여기서 $t_{\text{fixed}}$는 고정 텍스트 부분이고, $m_i$는 BERT의 `[MASK]` 토큰으로 설정된 최적화 대상이다.

핵심적으로, DEXTER는 소프트 프롬프트를 최적화하여 이를 이산적인 하드 프롬프트(discrete hard prompt)로 매핑함으로써 디퓨전 기반 이미지 생성 프로세스를 유도한다. 이는 생성된 이미지가 목표 분류기의 출력과 정렬되도록 보장하여, 모델이 우선시하는 특징과 개념을 포착한다.

활성화 최대화의 목적함수는 일반적으로 다음과 같이 표현된다:

```math
t^* = \arg\max_{t} \; f_c\bigl(G(t)\bigr)
```

- $t$: 텍스트 프롬프트 (소프트 프롬프트 → 하드 프롬프트로 매핑)
- $G(t)$: 프롬프트 $t$로 조건화된 스테이블 디퓨전의 이미지 생성 함수
- $f_c(\cdot)$: 목표 분류기의 클래스 $c$에 대한 활성화 점수 (예: 분류 헤드의 로짓 또는 소프트맥스 출력)

소프트 프롬프트 임베딩을 $e_s$라 하면, BERT MLM(Masked Language Model) 헤드를 통해 이산 토큰으로의 투영:

```math
m_i^* = \arg\max_{w \in \mathcal{V}} \; \text{BERT MLM}(e_s)_i \cdot \mathbf{1}_w
```

- $\mathcal{V}$: 어휘 집합(vocabulary)
- $\mathbf{1}_w$: 단어 $w$에 대한 원-핫 벡터

#### 2.2.3 이미지 생성 (Vision Pipeline)

스테이블 디퓨전을 통한 클래스 조건부 이미지 생성:

```math
x_0 = \text{SD}(t^*, \epsilon, T) = \text{Denoise}_T(\epsilon; \text{CLIP enc}(t^*))
```

- $\epsilon \sim \mathcal{N}(0, I)$: 초기 가우시안 노이즈
- $T$: 디노이징 타임스텝 수
- $\text{CLIP enc}(\cdot)$: CLIP 텍스트 인코더

전체 최적화 루프:

$$\mathcal{L} = -f_c\bigl(\text{SD}(t, \epsilon, T)\bigr) + \lambda \cdot \mathcal{R}(t)$$

- $\mathcal{R}(t)$: 텍스트 일관성 및 자연어 의미를 보존하기 위한 정규화 항
- $\lambda$: 정규화 가중치

#### 2.2.4 추론 모듈 (VLM Reasoning)

생성된 이미지들은 VLM에 의해 분석되며, VLM은 여러 이미지에 걸쳐 추론하여 모델의 의사결정 과정에 대한 일관되고 인간이 읽을 수 있는 텍스트 설명을 제공한다.

텍스트 보고서 생성:

$$R_c = \text{VLM}\bigl(\{x_0^{(1)}, x_0^{(2)}, \ldots, x_0^{(K)}\}; \text{prompt}_{\text{instruct}}\bigr)$$

- $R_c$: 클래스 $c$에 대한 자연어 설명 보고서
- $K$: 생성된 이미지 수

### 2.3 모델 구조

| 구성 요소 | 사용 모델 | 역할 |
|:---:|:---:|:---:|
| 텍스트 파이프라인 | BERT (MLM) | 소프트 → 하드 프롬프트 변환 |
| 이미지 생성 | Stable Diffusion | 클래스 조건부 이미지 합성 |
| 텍스트 조건화 | CLIP 텍스트 인코더 | 프롬프트-이미지 정렬 |
| 추론 모듈 | GPT-4o / LLaVA 등 VLM | 이미지 기반 텍스트 설명 생성 |
| 목표 모델 | RobustResNet50 등 | 설명 대상 분류기 |

### 2.4 성능 향상

ImageNet, Waterbirds, CelebA, FairFaces에 대한 실험을 통해 DEXTER가 전역적 모델 설명 및 클래스 수준 편향 보고에서 기존 접근법을 능가함을 확인했다.

정량적 평가에서 DEXTER는 CLIP-IQA 및 Semantic CLIP-IQA 지표에서 DiffExplainer보다 높은 점수 (0.94±0.03 / 0.96±0.03 vs. 0.89±0.09 / 0.89±0.09)를 달성하여 더 나은 의미적 정렬과 일관성을 보였다.

DEXTER는 다양한 왜곡에도 클래스 관련 특징을 일관되게 복원하는 견고성을 보였으며, 생성된 텍스트 보고서에서 추출한 핵심 시각적 단서를 이미지 프롬프트에 추가했을 때 분류기 활성화가 증가하여 텍스트 설명이 모델 관련 특징을 반영함을 확인했다.

### 2.5 한계점

논문과 공개된 정보에 기반한 한계점을 정리하면:

1. **외부 모델 의존성**: 고품질 VLM(GPT-4o 등)에 의존하므로 API 비용 및 접근성 문제가 존재한다.
2. **Diffusion 모델 제약**: Stable Diffusion이 생성할 수 없는 도메인(예: 의료 영상, 위성 이미지)에서는 적용이 제한될 수 있다.
3. **LLM 환각(Hallucination)**: VLM이 이미지로부터 존재하지 않는 특징을 설명할 가능성이 있다 (단, 논문에서는 텍스트 설명이 모델 관련 특징을 반영하며 환각이 아님을 통계적으로 검증했다).
4. **최적화 비용**: 소프트 프롬프트 최적화가 반복적인 디퓨전 추론을 수반하므로 계산 비용이 높다.
5. **텍스트 공간 한계**: 분류기가 학습한 특징이 자연어로 표현 불가능한 저수준 패턴일 경우 설명의 완전성이 저하될 수 있다.

## 3. 모델의 일반화 성능 향상 가능성

### 3.1 데이터 불필요(Data-Free) 특성의 일반화 기여

DEXTER는 인간이 해석할 수 있는 텍스트 설명을 생성하며, 디퓨전 모델의 생성 능력과 LLM의 추론 능력을 활용하여 **완전 데이터 불필요 환경**에서 작동한다.

이는 학습 데이터에 접근하지 않고도 편향을 발견할 수 있어, 데이터 프라이버시 제약이 있는 의료·법률 도메인 등에서의 일반화 적용을 가능하게 한다.

### 3.2 슬라이스 탐색을 통한 일반화 향상

Waterbirds 데이터셋(육상조류/수상조류 두 클래스)에서 대부분의 육상조류 샘플은 육지에, 수상조류는 물 근처에 있다. DEXTER는 새가 자연 서식지가 아닌 곳(물 위의 육상조류, 육지 위의 수상조류)에 있는 소수 샘플에서도 높은 성능을 보장하기 위해 편향된 슬라이스를 탐색한다.

이 과정에서 DEXTER는 분류기가 **배경(spurious feature)**에 의존하고 있음을 텍스트로 설명하고, 이를 기반으로 편향 제거(debiasing)를 수행함으로써 Worst-Group Accuracy를 개선한다.

### 3.3 편향 발견을 통한 일반화 메커니즘

이 접근 방식은 해석 가능성을 향상시키고, 사용자가 허위 상관관계(예: dog sled 클래스에서 5개의 핵심 특징 중 어느 것도 실제 sled를 포함하지 않음)를 식별하거나 분류기의 의미 있고 과업 관련 속성에의 의존을 확인할 수 있게 한다.

일반화 향상을 위한 DEXTER의 기여를 수식으로 표현하면:

**Worst-Group Accuracy 향상 목표:**

$$\max_\theta \; \min_{g \in \mathcal{G}} \; \mathbb{E}_{(x,y) \sim P_g} [\mathbf{1}(f_\theta(x) = y)]$$

- $\mathcal{G}$: 데이터 내 그룹(슬라이스) 집합
- $P_g$: 그룹 $g$의 분포
- DEXTER는 $\mathcal{G}$ 자체를 **데이터 없이 텍스트로 발견**함으로써 위 최적화를 지원한다.

기존의 데이터 의존적 방법과 달리, DEXTER는 분류기의 내부 동작에만 의존하여 **완전한 데이터 불필요 방식으로 편향을 발견하고 설명**한다.

## 4. 관련 최신 연구 비교 분석 (2020년 이후)

| 연구 | 연도 | 접근법 | 데이터 필요 | 설명 형태 | 편향 발견 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| **GradCAM** | 2020 | 그래디언트 기반 | ✅ 필요 | 시각적 히트맵 | ❌ |
| **Integrated Gradients** | 2020 | 그래디언트 적분 | ✅ 필요 | 픽셀 어트리뷰션 | ❌ |
| **B2T** (Bias-to-Text) | 2023 | 오분류 캡션 추출 | ✅ 필요 | 텍스트 | ✅ |
| **LADDER** | 2024 | LLM 편향 가설 생성 | ✅ 필요 | 텍스트+클러스터 | ✅ |
| **DiffExplainer** | 2024 | 디퓨전 활성화 시각화 | ❌ 불필요 | 이미지 | ❌ |
| **DEXTER** | 2025 | 소프트 프롬프트+디퓨전+VLM | ❌ 불필요 | **텍스트+이미지** | ✅ |

B2T는 오분류된 이미지 캡션에서 pseudo-bias 레이블을 추출하고, LADDER는 낮은 신뢰도 예측에서 편향 가설을 생성하며 LLM 파생 pseudo-attribute를 통해 샘플을 클러스터링한다.

GradCAM, Integrated Gradients 등의 기존 방법은 로컬 설명만 제공하고 데이터를 필요로 하며, 특정 픽셀이나 영역에 중요도를 귀속시키는 데 집중한다.

DEXTER가 이들과 차별화되는 핵심 우위는:

1. **완전 데이터 불필요** + **전역적 설명** 동시 달성
2. **자연어 텍스트** 형태의 설명 (비전문가도 이해 가능)
3. **편향 발견 → 편향 제거(debiasing)**까지의 엔드-투-엔드 파이프라인

## 5. 앞으로의 연구에 미치는 영향과 고려할 점

### 5.1 앞으로의 연구에 미치는 영향

**① XAI(Explainable AI) 패러다임 전환**

DEXTER는 기존의 **픽셀 수준 시각적 설명** 중심의 XAI 연구를 **자연어 기반 전역 설명**으로 전환하는 중요한 이정표가 된다. 향후 XAI 연구는 인간 친화적 텍스트 설명을 핵심 출력으로 삼는 방향으로 발전할 가능성이 높다.

**② 데이터 불필요 모델 분석의 확장**

DEXTER는 완전한 데이터 불필요 환경에서 작동하며, SalientImageNet, Waterbirds, CelebA, FairFaces에 대한 실험에서 허위 특징 및 핵심 특징 발견, 데이터셋 하위 집단 식별, 편향 설명 생성에서 효과성을 입증했다.

이는 데이터 프라이버시 규정(GDPR 등)이 강화되는 환경에서 특히 의료·금융 AI 모델의 감사(audit) 도구로 활용될 수 있다.

**③ Diffusion + LLM 융합 연구 촉진**

생성 모델(Diffusion)과 대형 언어 모델(LLM)을 해석 가능성 목적으로 결합하는 새로운 연구 방향을 제시하며, 멀티모달 XAI 연구의 기반을 마련한다.

**④ 공정성(Fairness) 연구와의 시너지**

자동화된 편향 발견 → 편향 제거 파이프라인은 AI 공정성 연구와 직접 연결된다. 향후 자동화된 모델 감사 시스템 개발에 핵심 구성요소로 활용될 가능성이 높다.

### 5.2 향후 연구 시 고려할 점

**① Diffusion 모델의 도메인 커버리지 확장**

현재 Stable Diffusion의 학습 도메인에 포함되지 않는 의료 영상(X-ray, MRI), 위성 이미지 등에 대한 적용 가능성을 높이기 위해 도메인 특화 Diffusion 모델을 결합하는 연구가 필요하다.

**② VLM 환각 문제의 체계적 억제**

생성된 텍스트 설명의 신뢰도 보장을 위해 VLM 환각을 정량적으로 측정하고 억제하는 메커니즘 연구가 필요하다. 예를 들어, 설명-활성화 일관성 검증 루프를 강화하는 방향을 고려할 수 있다.

**③ 최적화 효율성 개선**

소프트 프롬프트 최적화가 반복적 디퓨전 추론을 수반하므로, 더 효율적인 프록시 모델이나 원-샷 최적화 기법 도입이 실용적 배포를 위해 중요하다.

**④ 비분류 태스크로의 확장**

현재 DEXTER는 주로 시각적 **분류기**를 대상으로 하므로, 객체 탐지, 세그멘테이션, 생성 모델 등 다양한 비전 태스크로의 확장 가능성을 탐색할 필요가 있다.

**⑤ 다국어·다문화 편향 발견**

LLM/VLM의 언어적 편향이 설명 내용에 영향을 미칠 수 있으므로, 다국어 환경에서의 편향 발견 일관성 검증이 필요하다.

**⑥ 인간 평가와의 정렬**

정량적·정성적 평가(사용자 연구 포함)를 통해 DEXTER가 정확하고 해석 가능한 결과물을 생성함을 보였지만, 더 광범위하고 다양한 사용자 그룹을 대상으로 한 인간 평가 연구가 설명의 실제 유용성을 검증하는 데 중요하다.
