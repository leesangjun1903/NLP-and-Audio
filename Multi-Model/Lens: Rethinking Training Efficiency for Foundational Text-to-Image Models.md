
# Lens: Rethinking Training Efficiency for Foundational Text-to-Image Models

> **논문 출처:** arXiv:2605.21573 (2026)
> **GitHub:** https://github.com/microsoft/Lens
> **저자:** Dong Chen, Fangyun Wei, Ziyu Wan, Dongdong Chen 外 (Microsoft Research)

---

## 1️⃣ 핵심 주장 및 주요 기여 요약

Lens는 3.8B 파라미터의 Text-to-Image(T2I) 모델로, 6B 이상의 파라미터를 갖는 최신 SOTA 모델들과 경쟁하거나 일부 벤치마크에서 이를 능가하는 성능을 달성하면서도, 훨씬 적은 훈련 컴퓨팅 자원을 요구합니다.

예를 들어, Lens는 Z-Image가 사용한 훈련 컴퓨팅의 약 **19.3%** 만을 필요로 합니다.

이 논문은 파운데이션 T2I 모델의 **훈련 시간 효율성(training-time efficiency)** 향상에 집중하며, 데이터 캡셔닝 전략, VAE 선택 기준, 언어 인코더 선택, RL 기반 포스트 트레이닝을 위한 훈련 데이터 구성 등 실제로 종종 간과되는 훈련 효율 요소들을 체계적으로 탐구합니다.

각 요소에 대해 정량적 분석을 포함한 통제된 어블레이션 연구를 제공하며, 이 전략들은 아키텍처 혁신이나 분산 시스템 최적화와 같은 기존의 훈련 가속 접근 방식에 **상보적(complementary)** 임을 강조합니다.

## 2️⃣ 해결하고자 하는 문제, 제안 방법, 모델 구조, 성능 향상 및 한계

### 🔴 해결하고자 하는 문제

기존 T2I 시스템들은 강력한 성능을 달성하기 위해 점점 더 **대형 모델, 방대한 훈련 데이터, 복잡한 시스템 설계**를 요구하는 추세이며, 이는 효율적인 파운데이션 모델 훈련의 필요성을 강하게 촉구합니다.

따라서 훈련 시간 효율성 향상은 단순히 모델 규모를 줄이는 것에 그치지 않고, **각 배치의 학습 가치를 높이고 훈련 전반에 걸쳐 수렴 속도를 가속화**하는 것을 동시에 요구합니다.

---

### 🟢 제안하는 방법

Lens의 훈련 효율성은 컴팩트한 모델 크기 외에 **두 가지 핵심 전략**에서 비롯됩니다.

#### 전략 1: 배치당 데이터 정보 밀도 최대화 (Data Information Density Maximization)

첫 번째로, 훈련 배치당 데이터 정보 밀도를 극대화합니다. 구체적으로 (i) GPT-4.1이 생성한 캡션(평균 약 **109단어**)이 포함된 800M 이미지-텍스트 쌍으로 구성된 **Lens-800M** 데이터셋으로 훈련하여 기존의 짧은 캡션보다 풍부한 의미론적 지도학습을 제공하고, (ii) 다양한 해상도와 종횡비의 이미지로 각 배치를 구성하여 각 최적화 스텝의 유효 시각적 커버리지를 확장합니다.

이를 수식으로 표현하면, 배치 $\mathcal{B}$에서의 정보 밀도는 아래와 같이 구성됩니다:

$$\mathcal{B} = \{(x_i, c_i, r_i)\}_{i=1}^{N}$$

여기서 $x_i$는 이미지, $c_i$는 GPT-4.1이 생성한 고밀도 캡션(평균 109 단어), $r_i \in \mathcal{R}$은 다양한 해상도 및 종횡비를 나타냅니다.

훈련 목표는 Rectified Flow 기반 확산 모델의 손실 함수로 표현됩니다:

$$\mathcal{L} = \mathbb{E}_{t, x_0, \epsilon}\left[\left\| v_\theta(x_t, t, c) - (x_0 - \epsilon) \right\|^2\right]$$

여기서 $x_t = (1-t)x_0 + t\epsilon$, $v_\theta$는 학습되는 velocity field, $c$는 텍스트 조건입니다.

#### 전략 2: 수렴 속도 가속화 (Convergence Acceleration)

두 번째로, 더 나은 잠재 표현을 제공하는 **Semantic VAE** 채택과 최적화를 가속하면서 다국어 일반화를 가능하게 하는 **강력한 언어 인코더** 채택을 통해 수렴 속도를 개선합니다.

언어 인코더의 다층(multi-layer) 특징 연결은 아래와 같이 표현됩니다:

$$\mathbf{f}_{\text{text}} = \text{Concat}\left[\mathbf{h}^{(1)}, \mathbf{h}^{(2)}, \ldots, \mathbf{h}^{(L)}\right]$$

여기서 $\mathbf{h}^{(l)}$은 언어 인코더 $l$-번째 레이어의 hidden state이며, 이를 통해 단일 레이어 특징 대비 더 풍부한 의미 표현이 확보됩니다.

#### 포스트 트레이닝: RL 기반 품질 향상

사전 훈련 이후, **분류 체계 기반 프롬프트(Lens-RL-8K)** 와 구조화된 보상 루브릭을 활용한 RL을 적용하여 아티팩트를 억제하고 시각적 품질을 향상시키며, 훈련이 필요 없는 시스템 프롬프트 탐색을 통한 Reasoner 모듈을 추가합니다.

RL 보상 최적화 목표:

$$\mathcal{J}(\theta) = \mathbb{E}_{c \sim \mathcal{P}, x \sim p_\theta(\cdot|c)}\left[r(x, c)\right]$$

여기서 $r(x, c)$는 시각적 품질, 텍스트 정렬, 아티팩트 억제를 평가하는 구조화된 보상 함수입니다.

### 🔵 모델 구조

디노이징 백본은 **48개의 MMDiT 블록**으로 구성됩니다. 각 블록은 노이즈가 있는 이미지 특징과 이전 MMDiT 블록에서 생성된 텍스트 특징의 연결을 입력으로 받아, 이미지 및 텍스트 모달리티에 대해 두 개의 분리된 브랜치로 처리합니다. 정규화 레이어로는 **RMSNorm**을 사용하고, 이미지 특징에는 **RoPE(Rotary Positional Embedding)** 를 적용합니다.

**48블록 MMDiT 디노이저**는 FLUX.2 잠재 공간을 활용하고, 연결된 다층 GPT-OSS 특징을 사용하여 더 강력한 프롬프트 추종 및 다국어 일반화를 가능하게 합니다.

텍스트 표현의 연결 결과는 이미지 잠재 변수와 동일한 차원으로 투영하기 위한 **선형 어댑터(linear adapter)** 를 통해 처리됩니다.

혼합 해상도 훈련을 통해 **1:2 ~ 2:1의 종횡비**와 **최대 1440×1440 해상도**까지의 추론을 지원합니다.

모델 구조 요약:

| 구성 요소 | 세부 내용 |
|---|---|
| 파라미터 수 | 3.8B |
| 디노이저 구조 | 48-block MMDiT |
| 잠재 공간 | FLUX.2 VAE (Semantic VAE) |
| 언어 인코더 | GPT-OSS (MoE, 20B-A3B), multi-layer 특징 연결 |
| 정규화 | RMSNorm |
| 위치 인코딩 | RoPE |
| 해상도 | 최대 1440×1440, 종횡비 1:2~2:1 |
| 포스트 트레이닝 | RL + Lens-RL-8K 프롬프트 + Reasoner 모듈 |
| 추론 속도 | H100 GPU 기준 $1024^2$ 이미지 3.15초 (20 denoising steps) |

### 🟡 성능 향상

Lens-Turbo(4-step 증류 변형)는 생성 시간을 **0.84초**로 단축시킵니다. 고밀도 캡션과 혼합 해상도/종횡비 사전훈련으로 데이터 정보 밀도를 향상하고, 신중하게 선택된 VAE와 언어 인코더 설계로 수렴을 가속화함으로써, Lens는 현저히 낮은 훈련 비용으로 강력한 생성 품질을 달성합니다.

비교 대상 모델:
Lens는 3.8B로 파라미터 수를 제한한 반면, Z-Image(6B), LongCat-Image(6B), FLUX.2(9B), Qwen-Image(20B), Hunyuan-Image-3.0(MoE, 80B) 등 최신 오픈소스 SOTA 모델들은 6B 이상의 규모에서 운영됩니다.

### 🔴 한계

모델은 연구 목적으로만 공개되었으며, 제품 또는 서비스 배포 용도로는 의도되지 않았습니다.

책임 있는 AI 고려사항이 개발 과정 전반에 걸쳐 반영되었으나, 훈련 데이터는 공공, 라이선스, 내부 데이터셋의 조합으로 구성되어 있으며 명확하게 식별 가능한 개인정보 제거 및 유해 콘텐츠 감소 처리가 이루어졌습니다.

추가적으로 논문 자체에서 명시적으로 기술된 한계는 현재 공개된 정보의 범위 내에서는 확인이 제한적이므로, 아래와 같은 구조적 한계를 추론할 수 있습니다:
- GPT-4.1 캡션 생성 의존성으로 인한 **캡션 품질 편향** 가능성
- 영어 데이터 기반 훈련에 의한 **다국어 성능의 간접적 일반화** 한계
- 훈련 데이터 구성이 공개되지 않아 **재현성 제약** 존재

## 3️⃣ 일반화 성능 향상 가능성 (Generalization)

Lens가 일반화 성능을 향상시키는 핵심 메커니즘은 다음과 같습니다:

#### (1) 다국어 일반화 (Multilingual Generalization)
언어 인코더 선택의 주요 고려 사항 중 하나는, 영어 전용 이미지-텍스트 쌍으로 훈련하면서도 **다른 언어로의 추론(inference)을 지원**하는 다국어 일반화 능력입니다.

이를 검증하기 위해 GPT-OSS (MoE, 20B-A3B)와 다양한 크기(0.6B, 1.7B, 4B)의 Qwen3를 포함한 4가지 언어 인코더를 비교 분석합니다.

#### (2) 해상도 및 종횡비 일반화
혼합 해상도 훈련(Mixed-resolution Training)을 통해 **1:2 ~ 2:1의 다양한 종횡비와 최대 1440×1440 해상도**까지 일반화된 추론을 가능하게 합니다.

이는 다음 수식으로 표현되는 **해상도-불변 위치 인코딩** 전략과 결합됩니다:

$$\text{RoPE}(i, j) = \text{RotMat}(\theta_i) \cdot \text{RotMat}(\theta_j)$$

여기서 $(i, j)$는 2D 패치 좌표이고, 훈련 시 다양한 해상도의 입력을 받아 추론 시 미학습 해상도에서도 일반화됩니다.

#### (3) 의미론적 풍부성을 통한 일반화
GPT-4.1이 생성한 평균 109단어의 고밀도 캡션은 기존의 짧은 캡션 대비 풍부한 의미론적 지도 신호를 제공하며, 이는 단순 객체 인식을 넘어 복잡한 장면 구성과 추상적 개념에 대한 일반화 능력 향상으로 이어집니다.

#### (4) RL 포스트 트레이닝에 의한 분포 외 일반화
RL 튜닝은 시각적 품질 향상 및 아티팩트 억제를 개선하며, 증류 버전인 Lens-Turbo는 빠른 4-step 생성을 지원합니다.

## 4️⃣ 앞으로의 연구에 미치는 영향과 고려할 점

### 📈 앞으로의 연구에 미치는 영향

1. **데이터 중심(Data-Centric) 패러다임의 전환**
   단순한 데이터 양 증가가 아니라, **캡션 품질 밀도**를 통한 정보 효율성을 높이는 방향이 새로운 설계 원칙으로 자리잡을 가능성이 높습니다.

2. **소형 고효율 모델 설계 트렌드 강화**
   3.8B 파라미터의 비교적 컴팩트한 규모에도 불구하고 이전 SOTA 모델들과 경쟁하거나 이를 능가하는 성능을 보임으로써, 무조건적인 스케일업보다는 **데이터와 아키텍처의 공동 최적화**가 중요하다는 방향성을 제시합니다.

3. **VAE 및 언어 인코더 설계 지침 제공**
   VAE 선택 기준, 언어 인코더 선택 등에 관한 통제된 어블레이션 연구와 정량적 분석은, 향후 T2I 파운데이션 모델을 구축하는 연구자들에게 **실행 가능한 통찰(actionable insights)** 을 제공합니다.

4. **RL 기반 포스트 트레이닝의 T2I 적용 확대**
   텍스트-이미지 도메인에서의 RL 포스트 트레이닝 방법론이 구체화됨으로써, 강화학습을 통한 품질 향상 연구가 더욱 활발해질 것으로 기대됩니다.

5. **관련 비교 연구의 기준점(Baseline) 역할**
   Stable Diffusion 3 시리즈의 MMDiT 스타일 아키텍처와 Rectified Flow 훈련을 계승하고, FLUX, SANA, HiDream-I1 등 최신 모델들과의 비교를 통해 Lens는 효율적 T2I 연구의 강력한 기준선이 될 것입니다.

### ⚠️ 앞으로 연구 시 고려할 점

| 고려 사항 | 설명 |
|---|---|
| **캡션 생성 비용** | GPT-4.1 기반 고밀도 캡션 생성은 비용이 크므로, 오픈소스 대안 모델의 캡션 품질 비교 연구 필요 |
| **다국어 훈련 데이터 확장** | 영어 전용 훈련 후 다국어 일반화에 의존하는 구조의 한계를 명확히 규명해야 함 |
| **RL 보상 함수 설계** | 구조화된 보상 루브릭의 세부 설계가 성능에 미치는 영향에 대한 심층 연구 필요 |
| **공정한 벤치마크 비교** | Reasoner 기반 프롬프트 재작성이 현대 T2I 시스템에서 표준 관행이 되고 있으므로, 공정한 비교를 위해 Reasoner 사용 유무에 따른 결과를 모두 보고해야 합니다. |
| **재현성** | 내부 데이터셋 사용으로 인한 완전한 재현 불가 문제 해결 필요 |
| **비용-성능 트레이드오프 분석** | Lens-800M 구축에 든 실제 비용 대비 훈련 효율성 이득의 명확한 정량화 필요 |

| **Z-Image (6B)** | 대규모 모델 | Lens가 19.3% 컴퓨팅으로 동등 성능 달성 |
| **Flow-GRPO (arXiv:2505.05470, 2025)** | Flow Matching 모델 RL 온라인 훈련 | Lens의 RL 포스트 트레이닝과 방향 일치 |

Imagen, Stable Diffusion, FLUX 등의 최신 T2I 모델들은 훈련 데이터 품질에 의해 성능이 근본적으로 제한되며, 웹 크롤링 및 합성 이미지 데이터셋은 저품질 또는 중복 샘플을 포함하여 시각적 충실도 저하, 불안정한 훈련, 비효율적 연산을 초래하므로 효과적인 데이터 선택이 효율성 향상에 매우 중요합니다. Lens는 이 문제를 **데이터 선택**이 아닌 **캡션 품질 밀도 향상**으로 해결한다는 점에서 차별화됩니다.
